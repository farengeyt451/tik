# Delegation Agreement

## Framework
4D Framework — Delegation (Step 2)

## Roles

| Responsibility | Owner |
|---|---|
| All code implementation | AI |
| Project structure & architecture | AI (proposed) → Human (approved) |
| Requirements & direction | Human |
| Visual acceptance testing | Human |
| Edge case discovery | Human |
| Final approval on all deliverables | Human |
| Iteration based on feedback | AI |

## How We Work

- AI drafts, Human reviews and redirects
- Human tests in the real terminal — AI cannot see the screen
- When output feels wrong visually, Human describes it; AI adjusts
- Human makes all judgment calls; AI handles execution
- Human focuses on **direction + review**, not coding

## Boundaries

- AI does not proceed to next implementation step without Human's explicit command
- AI proposes; Human decides on any ambiguous UX or architecture choice
- Feedback flows one way per cycle: Human reviews → AI iterates
